Features implemented

  • Week 1 (20 points)
    • (10 points) Face detection
    • (10 points) Individual feature detection (eyes, nose, mouth)
  • Week 2 (20 points)
    • (20 points) Normalized poses for each face in the dataset. Use generic face model to transform found features to a neutral pose.
      • We did not implement this as described in the method section of the paper. Doing so would have required the implementation of the system in the paper, Direct Visibility of Point Sets, along with a 3d model loader and a simple rendering system.
      • Instead we implemented the technique presented in the conclusion of the paper. The authors note that using 2d affine transformations to warp to neutral poses was almost as effective as the 3d approach glossed over in the method.
  • Week 3 (20 points)
    • (10 points) Implement the similarity metric
    • (10 points) Build face graph
  • Week 4 (15 points)
    • (10 points) Interpolate between given initial and final faces Find inbetween images using the shortest path between faces
    • (5 points) Create a video that cross dissolves between the returned sequence of images

Using the system

  • System requirements:
    • python-opencv
    • numpy
    • ffmpeg (must be in $PATH)
    • ImageMagick (must be in $PATH)

  • Running the preprocessor: 
    ./preprocessor.py rip/info.pickle rip/used rip/faces rip/source/*
  • Building the face graph: 
    ./mkgraph.py rip/info.pickle rip/graph.pickle 1
    The final argument is the exponent that each edge is raised to. This is intended to control the length of the path returned during interpolation.

  • Interpolating: 
    ./interp-faces.py info.pickle graph.pickle rip/used/0.jpg rip/used/100.jpg depth frames working/ output.avi
    The depth argument is the number of expansions to run on the returned path. A value of 1 returns the best results. The frames argument is the number of frames to use for in transitioning between two images.
